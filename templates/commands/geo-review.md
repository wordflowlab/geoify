# GEO 审校

我将对你的内容进行 GEO 专项审校,重点评估 E-E-A-T 维度并提供优化建议。

## 审校维度

### 1. Experience (体验) - 8/10
**评估重点**:
- ✅ 是否基于真实经历
- ✅ 是否有具体细节支撑
- ✅ 是否避免泛泛而谈

**优化方向**:
- 添加个人实践案例
- 补充具体数据/时间
- 避免"据说"、"可能"等模糊表述

### 2. Expertise (专业性) - 8/10
**评估重点**:
- ✅ 是否展示专业知识
- ✅ 是否有深度分析
- ✅ 是否避免常识性内容

**优化方向**:
- 添加技术细节
- 补充行业术语解释
- 提供独特见解

### 3. Authoritativeness (权威性) - 8/10
**评估重点**:
- ✅ 是否引用权威来源
- ✅ 是否有外部认可
- ✅ 是否建立作者可信度

**优化方向**:
- 添加权威数据来源引用
- 补充作者资质说明
- 链接到相关权威资源

### 4. Trustworthiness (可信度) - 9/10
**评估重点**:
- ✅ 数据是否可验证
- ✅ 来源是否明确
- ✅ 是否有联系方式

**优化方向**:
- 所有数据添加出处
- 补充发布/更新时间
- 添加作者联系方式

## 审校流程

### 第 1 遍:事实核查
- 检查所有声明是否有依据
- 验证数据准确性
- 标注需要补充来源的内容

### 第 2 遍:E-E-A-T 评分
- 对 4 个维度独立评分
- 识别薄弱环节
- 生成优化建议清单

### 第 3 遍:GEO 格式优化
- 优化标题和结构
- 添加结构化数据标记
- 优化引用格式

## 输出内容

审校完成后,生成:

```
articles/[话题名称]/
├── review-report.md      # 审校报告
├── eeat-score.json       # E-E-A-T 评分详情
└── optimization-list.md  # 优化建议清单
```

### 评分示例
```json
{
  "overall": 8.25,
  "breakdown": {
    "experience": 8,
    "expertise": 8,
    "authoritativeness": 8,
    "trustworthiness": 9
  },
  "recommendations": [
    "添加 3 个具体案例",
    "补充 5 个数据来源引用",
    "添加作者简介"
  ]
}
```
