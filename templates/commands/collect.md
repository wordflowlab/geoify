# 素材收集

我将帮你收集真实数据和案例,建立可信的素材库。

## 为什么需要素材库?

**E-E-A-T 的核心**: 内容必须基于真实数据和经历,而非凭空编造。

素材库帮你:
- ✅ 提供可验证的数据
- ✅ 丰富真实案例
- ✅ 建立权威引用
- ✅ 降低 AI 味

## 第一步:明确素材需求

请告诉我:

1. **内容主题**
   - 例如: "Python vs JavaScript 初学者对比"

2. **需要的素材类型**
   - 数据统计
   - 真实案例
   - 专家观点
   - 测试结果
   - 用户反馈

## 第二步:素材类型

### 1. 数据统计
**来源**:
- 官方统计报告
- 行业调研数据
- 学术研究结果

**示例**:
- Stack Overflow Developer Survey
- GitHub Octoverse Report
- TIOBE Index

**收集内容**:
- 数据来源 URL
- 发布时间
- 关键数据点
- 引用格式

### 2. 真实案例
**来源**:
- 你的亲身经历
- 客户案例
- 公开成功故事

**示例**:
- "我们团队从 Trello 迁移到 Linear 的过程"
- "某创业公司 3 个月节省 30% 成本"

**收集内容**:
- 背景信息
- 具体数据 (时间、数字)
- 结果和影响
- 可验证证据

### 3. 专家观点
**来源**:
- 行业专家访谈
- 权威博客文章
- 技术演讲

**示例**:
- "Google 工程师对 Python 未来的看法"
- "知名产品经理的工具选择建议"

**收集内容**:
- 专家背景资质
- 观点原文引用
- 来源链接
- 发布时间

### 4. 测试结果
**来源**:
- 你的实际测试
- 第三方评测
- 性能基准测试

**示例**:
- "10 个项目管理工具速度对比"
- "Python vs JavaScript 执行效率测试"

**收集内容**:
- 测试方法
- 测试环境
- 原始数据
- 测试截图

### 5. 用户反馈
**来源**:
- 用户调研
- 社交媒体讨论
- 评论和评价

**示例**:
- Reddit 讨论精华
- Twitter 用户反馈
- G2/Capterra 评价

**收集内容**:
- 用户身份 (匿名化)
- 原始反馈
- 来源链接
- 收集时间

## 第三步:素材组织

我会帮你将素材整理到:

```
materials/
├── raw/                      # 原始素材
│   ├── data/                 # 数据统计
│   ├── cases/                # 真实案例
│   ├── expert-quotes/        # 专家观点
│   ├── tests/                # 测试结果
│   └── feedback/             # 用户反馈
│
├── indexed/                  # 主题索引
│   └── [话题名称]/
│       ├── sources.json      # 来源清单
│       └── citations.md      # 引用格式
│
└── archive/                  # 历史素材
    └── outdated/             # 过时数据
```

### sources.json 示例

```json
{
  "topic": "Python vs JavaScript 2024",
  "collected": "2024-01-15",
  "sources": [
    {
      "type": "data",
      "title": "Stack Overflow Developer Survey 2024",
      "url": "https://survey.stackoverflow.co/2024",
      "date": "2024-01-10",
      "key_findings": [
        "Python: 49.3% 开发者使用",
        "JavaScript: 63.6% 开发者使用"
      ],
      "citation": "Stack Overflow. (2024). Developer Survey 2024."
    },
    {
      "type": "case",
      "title": "我们团队的技术选型经历",
      "source": "personal",
      "date": "2023-12-01",
      "summary": "3 人创业团队选择 Python 的原因和结果",
      "data_points": [
        "开发时间缩短 40%",
        "部署成本降低 30%"
      ]
    }
  ]
}
```

## 第四步:素材验证

我会帮你检查:

### 1. 真实性
- ✅ 来源可追溯
- ✅ 数据可验证
- ✅ 时间明确

### 2. 权威性
- ✅ 来源可信
- ✅ 作者有资质
- ✅ 发布平台权威

### 3. 时效性
- ✅ 数据最新
- ✅ 标注发布时间
- ✅ 定期更新

## 第五步:引用格式生成

为每个素材生成标准引用格式:

**学术格式**:
```
Stack Overflow. (2024). Developer Survey 2024.
Retrieved from https://survey.stackoverflow.co/2024
```

**内容中引用**:
```
根据 Stack Overflow 2024 年开发者调查,Python 被 49.3% 的开发者使用[1]。

[1] https://survey.stackoverflow.co/2024
```
